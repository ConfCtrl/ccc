About the diploma porject

The goal of this diploma project is, to simplify the network administration by monitoring and saving data.

Monitoring network devices

Monitoring network devices is defined as regular requests on various "show commands" of the devices, to detect possible changes, failures or performance problems such as missing disk space, and inform the network administrator. Which devices have to be monitored, is declared by the network administrator on the first configuration and can be changed every time. Furthermore, we took a special focus on VoIP devices (Cisco Phones). The reachability of these VoIP devices is checked regularly and the network administrator will be informed in case of failure. Which devices have to be monitored, is declared by the network administrator on the first configuration and can be changed every time.

Configuration backub of network devices

Configuration backup is defined as regular saving of the "Running-Configuration" and other relevant "show commands"(what commands are relevant is defined and tested during the diploma project). These informations are saved local on the computer our program/tool is running on. Our program/tool creates a suitable folder structure for this. This regular savings are tested on their consistency and in case of a change in the configuration file compared to the last saved configuration file, a status report is sent to the administrator.
